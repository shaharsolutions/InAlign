ניהול זהויות ואימות

המערכת מתבססת על Supabase Auth לניהול משתמשים ואימותם בצורה ריכוזית ומאובטחת.

פרוטוקול JWT

שימוש בטוקנים חתומים ומוצפנים לניהול סשנים של משתמשים.

הצפנת סיסמאות

מנגנון Hashing מתקדם (BCrypt) המופעל ע"י Supabase.

בקרת גישה והרשאות (RLS)

אבטחת המידע נאכפת ישירות ברמת מסד הנתונים באמצעות Row Level Security.

Multi-Tenancy (בידוד ארגוני)

הפרדה מוחלטת בגישה לנתונים - מנהלי ארגון רואים רק את המשתמשים והתכנים המשויכים אליהם.

מודל תפקידים (RBAC)

מנגנון הרשאות מדורג: Super Admin, Org Admin ו-Learner.

אבטחת נתונים ותכנים

אחסון מאובטח

קבצי לומדות מאוחסנים ב-Storage מוגן המאפשר גישה לפי הרשאות בלבד.

הצפנה במעבר

כל התקשורת מתבצעת תחת פרוטוקול HTTPS/TLS 1.3.

המלצות לחיזוק האבטחה

1. אימות דו-שלבי (MFA)

מומלץ להפעיל MFA לחשבונות מנהלי על ומנהלי ארגון.

2. Content Security Policy (CSP)

הטמעת כותרות CSP למניעת הזרקת קוד זדוני ממקורות חיצוניים.

3. Rate Limiting

הגדרת מגבלות קצב בקשות למניעת ניסיונות Brute Force.
